# Questions I ask myself before starting any project

During this part of the project (at the very beginning), I organize my guiding questions into categories such as problem, user, purpose, scope, data, uniqueness, draft, and future growth.

**Problem**

- What is the problem I'm trying to solve?
- Why does this problem matter?
- Why does this problem exist?
- What happens if the problem is not solved?

**User**

- Who is the project for?
- What are some of the things user strugge with right now?
- What should the user be able to do easily?
- What makes my solution very appealing to the users?
  
**Purpose**

- Why am I building this project?
- What skill, idea, and knoweledge realted to programming will this project test and prove?
- What should this project say about me as a developer?
- What will I discover about myself during this specfic project, which I can't in other projects?
  
**Scope**

- What is the smallest, concise useful version which I can build first? (the essence)
- What are the main features of the project?
- What should I avoid?
- What's the limit to how effiecnet, condensed, minimum error can the code can be?

**Data**

- What data would be program need?
- Where will the data come from?
- How should the data be stored, organized, and manipulated?
- How can I use only the essential data needed to generate the most useful output in the context of this prohect?

**Uniqueness**

- What makes this project different?
- What are the program's interesting features (wow factor)?
- How can I balance creativity with practicality?
- Should the project focus more on visual design, interaction, intelligence, or personalization?

**Draft**

- What can the frist draft version look like?
- In which order should I add the features?
- What are the things which I will produly show even if my project remains incomplete?

**Future growth**
- How can this project grow after the MVP?
- What features can be added later?
- Is this portfolio a case study, product, or content?
